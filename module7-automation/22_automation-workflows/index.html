    第二十二章：自動化工作流實作¶

    案例一：每日郵件摘要 Agent¶

    把重複性的工作交給自動化腳本，是 Claude API 最實用的應用場景之一。這個案例會帶你建立一個每天早上自動整理郵件的 Agent。

    架構¶

    整體流程相當直觀：

    每天早上 8:00
    │
    ▼
AppleScript 讀取 Outlook 郵件
    │
    ▼
Python 呼叫 Claude API（搭配 email-summary skill）
    │
    ▼
生成 HTML 報告
    │
    ▼
自動開啟瀏覽器顯示

    實作¶

    # email_agent.py
import anthropic
import subprocess
import os
from datetime import datetime

def get_emails_via_applescript():
    """使用 AppleScript 讀取 Outlook 郵件（僅 macOS）"""
    script = '''
    tell application "Microsoft Outlook"
        set today_emails to {}
        set inbox to inbox
        set msgs to messages of inbox
        set result_text to ""
        repeat with msg in msgs
            set msg_date to time received of msg
            -- 只取今天的郵件
            set result_text to result_text & "寄件者：" & (sender of msg as string) & "\\n"
            set result_text to result_text & "主旨：" & subject of msg & "\\n"
            set result_text to result_text & "---\\n"
        end repeat
        return result_text
    end tell
    '''
    result = subprocess.run(['osascript', '-e', script], 
                          capture_output=True, text=True)
    return result.stdout

def summarize_emails(email_text):
    """使用 Claude API 摘要郵件"""

    # 載入 email-summary skill
    skill_path = os.path.expanduser('~/.claude/skills/email-summary/SKILL.md')
    with open(skill_path, 'r') as f:
        skill_content = f.read()

    client = anthropic.Anthropic()

    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=2048,
        system=f"""你是郵件摘要助手。

請遵循以下規範：
{skill_content}
""",
        messages=[
            {
                "role": "user",
                "content": f"請摘要今天的郵件：\n\n{email_text}"
            }
        ]
    )
    return response.content[0].text

def generate_html_report(summary):
    """生成 HTML 報告"""
    today = datetime.now().strftime('%Y/%m/%d')
    html = f"""<!DOCTYPE html>
<html lang="zh-TW">
<head>
    <meta charset="UTF-8">
    <title>每日郵件摘要 {today}</title>
    <style>
        body {{ font-family: -apple-system, sans-serif; max-width: 800px; margin: 40px auto; }}
        h1 {{ color: #333; }}
        .summary {{ line-height: 1.8; white-space: pre-wrap; }}
    </style>
</head>
<body>
    <h1>📧 每日郵件摘要 — {today}</h1>
    <div class="summary">{summary}</div>
</body>
</html>"""

    output_path = f"output/email_summary_{datetime.now().strftime('%Y%m%d')}.html"
    os.makedirs('output', exist_ok=True)
    with open(output_path, 'w') as f:
        f.write(html)
    return output_path

if __name__ == "__main__":
    emails = get_emails_via_applescript()
    summary = summarize_emails(emails)
    report_path = generate_html_report(summary)
    subprocess.run(['open', report_path])
    print(f"✅ 報告已生成：{report_path}")

    定時執行設定（macOS launchd）¶

    腳本寫好後，用 launchd 設定定時自動執行。

    Windows 用戶：讀取 Outlook 郵件請改用 PowerShell COM 自動化或 Microsoft Graph API；定時執行請改用「工作排程器（Task Scheduler）」；open report_path 請改成 subprocess.run(['start', report_path], shell=True)。完整範例見 Windows 差異說明。

    # ~/Library/LaunchAgents/com.user.email-agent.plist
cat > ~/Library/LaunchAgents/com.user.email-agent.plist << 'PLIST'

<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "...">
<plist version="1.0">
<dict>
    <key>Label</key>

    <key>ProgramArguments</key>

        <string>/usr/bin/python3</string>

    案例二：文件批次分析¶

    需要分析大量文件時，Batch Processing 是最省成本的方式：

    # 批次分析資料夾中所有 PDF 的摘要
import anthropic
import os

client = anthropic.Anthropic()

def batch_analyze_documents(folder_path):
    requests = []
    for i, filename in enumerate(os.listdir(folder_path)):
        if filename.endswith('.txt'):
            with open(os.path.join(folder_path, filename)) as f:
                content = f.read()
            requests.append({
                "custom_id": f"doc-{i}-{filename}",
                "params": {
                    "model": "claude-haiku-4-5-20251001",
                    "max_tokens": 512,
                    "messages": [{
                        "role": "user",
                        "content": f"請用 3 點摘要這份文件：\n\n{content[:3000]}"
                    }]
                }
            })

    batch = client.messages.batches.create(requests=requests)
    print(f"批次建立完成，ID：{batch.id}")
    return batch.id

    實作練習¶

    1. 簡化版郵件摘要：不使用 AppleScript，改用一個純文字的 emails.txt 作為輸入，實作 summarize_emails() 和 generate_html_report() 兩個函數，執行後在瀏覽器中確認 HTML 報告正常顯示。

    2. 加入錯誤處理：在 email_agent.py 的 summarize_emails() 函數中加入 try/except，處理以下情境：API Key 未設定、網路超時、郵件內容為空。確保任何一個步驟失敗時，程式能印出有用的錯誤訊息而不是直接崩潰。

    3. 設定定時執行：依照本章的 launchd 設定範例，建立 com.user.email-agent.plist 並設定在每天早上 8:00 執行。用 launchctl load 載入後，用 launchctl list | grep email-agent 確認已成功排程。

    常見問題 Q&A¶

    Q：AppleScript 讀取 Outlook 郵件時顯示「Not permitted」，怎麼辦？ A：macOS 需要給予 Terminal（或你的 Python 執行環境）存取 Outlook 的權限。前往「系統設定 → 隱私與安全性 → 自動化」，確認你的終端機應用程式有勾選 Microsoft Outlook 的存取權限。

    Q：launchd 的 plist 設定完成後，如何確認定時任務在運作？ A：執行 launchctl list com.user.email-agent 確認任務已載入（會顯示 PID 或 0）。首次測試可臨時用 launchctl start com.user.email-agent 立即觸發一次，確認腳本能正常執行。

    Q：Batch Processing 的結果怎麼取回？ A：建立批次後記錄 batch.id，定期執行 client.messages.batches.retrieve(batch_id) 查詢狀態。狀態變成 ended 後，用 client.messages.batches.results(batch_id) 取回所有結果，或下載至 JSONL 檔案。

    第十三章：Skills 核心概念¶

    什麼是 Skills？¶

    Skills 是 Claude Code 的可重複使用指令模組系統，於 2025 年 10 月由 Anthropic 正式推出，並在 2025 年 12 月成為 agentskills.io 開放標準。

    最簡單的定義¶

    Skill = 一個資料夾 + 一個 SKILL.md 檔案

    Skills 解決什麼問題？¶

    你有沒有過這樣的體驗：每次叫 Claude 寫 commit，都要重新說一次「請用這種格式」；每次做程式碼審查，都要貼上同一份 checklist。Skills 就是把這些「每次都要說的事」寫進一個檔案，讓 Claude 自己去讀、自己去用。

    沒有 Skills：
  你每次都要告訴 Claude「我的 git commit 格式是...」
  你每次都要貼上程式碼規範

有了 Skills：
  Claude 自動偵測任務類型 → 自動載入對應 Skill → 直接按規範執行

    Skills 的結構¶

    一個 Skill 就是一個小資料夾，核心是 SKILL.md 這個指令檔，其他都是選配：

    my-skill/
├── SKILL.md          ← 必要：指令核心
├── examples/         ← 選用：範例檔案
│   └── example.py
└── templates/        ← 選用：模板檔案
    └── template.md

    SKILL.md 結構¶

    ---
name: skill-name
description: 一句話說明何時觸發（這非常重要！）
---

# Skill 標題

## 說明
這個 Skill 的用途說明

## 執行步驟
1. 步驟一
2. 步驟二

## 格式規範
- 規範一
- 規範二

## 範例
[範例內容]

    三種觸發方式¶

    Skills 可以用三種方式啟動，各有適合的場景：

    方式 說明 範例
    自動觸發 Claude 根據訊息內容自動偵測 說「幫我 commit」→ 自動載入 git-commit skill
    斜線指令 用 /skill名稱 明確呼叫 /code-review
    明確要求 直接說要使用某 Skill 「使用 debug skill 分析這個錯誤」

    Skills vs MCP vs Subagent¶

    這三個概念容易混淆，記住一個關鍵：Skills 是「說明怎麼做」、MCP 是「提供能力」、Subagent 是「分派工作」。

    Skills
  → 指令集，告訴 Claude「要怎麼做」
  → 存放於 SKILL.md 文字檔
  → 影響主要 Agent 的行為
  → 範例：「commit 訊息要遵循 Conventional Commits 格式」

MCP（Model Context Protocol）
  → 工具集，給 Claude「能做什麼」的新能力
  → 存放於 Server 程式（Node.js/Python）
  → 提供外部系統存取能力
  → 範例：「可以讀取本地檔案系統」

Subagent
  → 獨立的工作者 Agent，接受主 Agent 委派任務
  → 平行執行互不相依的任務
  → 範例：「同時測試三個不同模組」

搭配使用：
Skills 定義「如何使用」MCP 工具
Skills 規定「何時委派」給 Subagent

    Skills 生態系（2026 現狀）¶

    • 官方 Anthropic Skills：內建於 Claude Code，所有方案可用
    • 社群 Skills：agentskills.io 上有數千個可下載
    • 自訂 Skills：自己撰寫，無額外費用
    • 跨工具相容：同一個 SKILL.md 可在 Claude Code、Cursor 等工具使用

    實作練習¶

    1. 查看已載入的 Skills：啟動 Claude Code，在對話中輸入「你目前有哪些 Skills 可以使用？請列出名稱和觸發時機」，了解預設狀態下有哪些內建 Skills。

    2. 三種觸發方式體驗：選擇一個內建 Skill（如 code-review），分別用三種方式觸發並比較差異：

    3. 自動觸發：「請幫我審查這段程式碼 [貼上程式碼]」
    4. 斜線指令：/code-review

    5. 明確要求：「請使用 code-review skill 分析這個函數」

    6. 概念對照練習：為以下需求判斷要用 Skills、MCP，還是 Subagent：

    7. 「每次 commit 都要檢查有沒有把 API key 寫進程式碼」→ ___
    8. 「讀取本地 SQLite 資料庫的內容」→ ___
    9. 「同時分析三個不同模組的效能瓶頸」→ ___

    常見問題 Q&A¶

    Q：Skills 需要安裝什麼依賴套件嗎？ A：不需要。Skills 只是 Markdown 文字檔（SKILL.md），不含任何可執行程式碼（Skills 2.0 除外）。Claude 讀取文字後按照內容行動，完全不需要 npm 或 pip 安裝任何東西。

    Q：一個 Skill 可以觸發另一個 Skill 嗎？ A：可以，只需要在 SKILL.md 的步驟中明確說明「完成後觸發 xxx-skill」，Claude 會依序執行。這是建立複合工作流的常見做法。

    Q：Skills 和 Claude.ai Custom Instructions 的差異是什麼？ A：Custom Instructions 在所有 Claude 產品中生效，適合個人偏好設定；Skills 只在 Claude Code（CLI）中生效，適合結構化的工作流程，且可以有多個 Skills 依任務類型自動切換。

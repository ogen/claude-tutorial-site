    第十四章：Skills 安裝路徑與作用範圍¶

    Skills 存放位置¶

    Skills 放在哪裡，決定了它在哪些地方生效。兩種位置各有用途：

    全域 Skills（User Scope）¶

    放在家目錄的 .claude/skills/，所有專案都能用，適合你個人的通用規範：

    ~/.claude/skills/
└── skill-name/
    └── SKILL.md

    專案 Skills（Project Scope）¶

    放在專案根目錄的 .claude/skills/，只有這個專案用得到，適合專案特定的規則：

    your-project/
└── .claude/
    └── skills/
        └── skill-name/
            └── SKILL.md

    建立目錄結構¶

    # 建立全域 skills 目錄
mkdir -p ~/.claude/skills

# 建立專案 skills 目錄
mkdir -p .claude/skills

    範圍選擇建議¶

    不確定要放哪裡的話，用這個表格當參考：

    Skill 類型 建議範圍 原因
    Git commit 格式 全域 所有專案都適用
    程式碼風格規範 專案 每個專案規範不同
    測試撰寫規則 專案 依框架而異
    文件格式標準 全域 個人一致風格
    公司特定規範 全域 跨專案共用

    安裝社群 Skill¶

    有幾種方法可以取得社群分享的 Skill：

    # 方法一：手動下載
mkdir -p ~/.claude/skills/frontend-design
curl -o ~/.claude/skills/frontend-design/SKILL.md \
  https://raw.githubusercontent.com/anthropics/skills/main/frontend-design/SKILL.md

# 方法二：git clone 整個 skills 倉庫
cd ~/.claude
git clone https://github.com/anthropics/claude-skills skills

# 方法三：Claude Code 自動安裝（若支援）
claude skills install frontend-design

    確認 Skills 已載入¶

    安裝完成後，重新啟動 Claude Code 對話，直接問它確認：

    # 在 Claude Code 對話中
/skills list

# 或詢問 Claude
> 你目前有哪些 Skills 可以使用？

    Skills 2.0 新功能（2026 Q1）¶

    Skills 2.0 新增可執行腳本支援，Skill 不再只是文字指令，還可以附帶實際可執行的腳本：

    my-skill/
├── SKILL.md
├── run_tests.sh      ← 可執行腳本
└── query_db.py       ← 資料庫查詢腳本

    讓 Skill 能直接執行測試套件、查詢資料庫，而不只是提供文字指令。

    實作練習¶

    1. 建立目錄結構：執行以下指令，同時建立全域與專案層級的 Skills 目錄，確認路徑正確： 

      mkdir -p ~/.claude/skills
mkdir -p .claude/skills
ls -la ~/.claude/skills
ls -la .claude/skills

    2. 安裝社群 Skill：依照本章的「方法一」手動下載一個社群 Skill。下載後重啟 Claude Code，詢問「你現在有哪些 Skills？」確認新 Skill 出現在清單中。

    3. 範圍決策練習：根據本章的範圍選擇建議表，為以下 Skill 決定應放在全域（~/.claude/skills/）還是專案（.claude/skills/）：

    4. 「commit message 統一使用 Conventional Commits 格式，語言為繁體中文」→ ___
    5. 「這個 Django 專案的 API 測試要按照 tests/README.md 的規範撰寫」→ ___
    6. 「所有文件標題層級遵循本公司的寫作風格指南」→ ___

    常見問題 Q&A¶

    Q：全域 Skills 和專案 Skills 同名時，哪個優先？ A：專案層級（.claude/skills/）優先。這讓你可以在全域設定通用規則，再於特定專案用同名 Skill 覆蓋，非常適合有特殊規範的大型專案。

    Q：Skills 可以版本控制嗎？ A：可以，而且強烈建議。將 .claude/skills/ 資料夾加入 git 追蹤，Skill 的修改記錄會保存在 git 歷史中，團隊成員 clone 專案後也能直接使用同一套 Skills。

    Q：claude skills install 指令目前可以用嗎？ A：此功能尚在實驗階段，部分版本的 Claude Code 支援，建議先用手動下載（方法一）或 git clone（方法二）確保相容性。

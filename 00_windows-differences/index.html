    Windows 使用者差異說明¶

    這份文件整理了本教學中所有 macOS 專用指令與 Windows 的對應做法。如果你是 Windows 用戶，建議先瀏覽一遍，遇到卡關時再回來對照。

    快速對照表¶

    章節 macOS Windows
    第 7、8 章 ~/Library/Application Support/Claude/ %APPDATA%\Claude\
    第 8 章 tail -f ~/Library/Logs/Claude/mcp*.log Get-Content "$env:APPDATA\Claude\Logs\mcp*.log" -Wait
    第 9 章 curl ... \| bash（安裝 nvm） 下載 nvm-windows 安裝程式
    第 9 章 ~/.zshrc / ~/.bashrc PowerShell Profile（$PROFILE）
    第 10 章 touch CLAUDE.md New-Item CLAUDE.md -ItemType File
    第 10 章 ~/.claude/CLAUDE.md C:\Users\你的用戶名\.claude\CLAUDE.md
    第 11 章 Cmd+Shift+X Ctrl+Shift+X
    第 11 章 Cmd+Shift+P Ctrl+Shift+P
    第 11 章 iTerm2 Windows Terminal
    第 18 章 export KEY=...（加入 ~/.zshrc） 系統環境變數（圖形介面）或 PowerShell
    第 22 章 AppleScript PowerShell COM / Microsoft Graph API
    第 22 章 launchd（plist 檔） 工作排程器（Task Scheduler）
    第 22 章 open output.html start output.html

    第 7–8 章：MCP 設定檔與日誌路徑¶

    設定檔位置¶

    macOS：~/Library/Application Support/Claude/claude_desktop_config.json
Windows：%APPDATA%\Claude\claude_desktop_config.json

    在 Windows 上用記事本或 VS Code 開啟：

    # 用 VS Code 開啟設定檔
code "$env:APPDATA\Claude\claude_desktop_config.json"

# 用記事本開啟
notepad "$env:APPDATA\Claude\claude_desktop_config.json"

    查看 MCP 日誌¶

    # Windows PowerShell — 即時追蹤日誌
Get-Content "$env:APPDATA\Claude\Logs\mcp-server-filesystem.log" -Wait

# 查看所有 MCP 日誌檔案
Get-ChildItem "$env:APPDATA\Claude\Logs\" -Filter "mcp-server-*.log"

    第 9 章：Node.js 與 Claude Code 安裝¶

    安裝 nvm（Windows 版）¶

    macOS 用的 nvm 腳本在 Windows 上無法使用，需要改用 nvm-windows：

    1. 前往 https://github.com/coreybutler/nvm-windows/releases
    2. 下載 nvm-setup.exe 並執行安裝
    3. 安裝完成後開啟新的 PowerShell 視窗，執行：
    nvm install 22
nvm use 22
node --version   # 確認版本

    安裝 Claude Code¶

    npm install -g @anthropic-ai/claude-code

    Windows 上安裝 npm 全域套件若出現權限問題，建議以系統管理員身分執行 PowerShell，或參考 npm 官方說明設定不需要 sudo 的全域路徑。

    第 10 章：基礎指令對照¶

    建立檔案¶

    # macOS
touch CLAUDE.md

# Windows PowerShell
New-Item CLAUDE.md -ItemType File
# 或縮寫
ni CLAUDE.md

    全域 CLAUDE.md 路徑¶

    macOS：~/.claude/CLAUDE.md
Windows：C:\Users\你的用戶名\.claude\CLAUDE.md

    路徑分隔符¶

    Windows 路徑使用反斜線（\），但 Claude Code CLI 在 Windows 上也接受正斜線（/），多數情況下可以直接使用。

    第 11 章：IDE 鍵盤快捷鍵¶

    VS Code 在 Windows 上把 Cmd 換成 Ctrl，其他完全相同：

    功能 macOS Windows
    開啟擴充功能市集 Cmd+Shift+X Ctrl+Shift+X
    開啟指令面板 Cmd+Shift+P Ctrl+Shift+P

    終端機推薦¶

    macOS 推薦 iTerm2，Windows 推薦使用 Windows Terminal（Microsoft Store 免費下載），支援分割視窗，使用體驗和 iTerm2 類似。

    第 18 章：API Key 環境變數設定¶

    方法一：圖形介面（簡單）¶

    1. 搜尋「環境變數」→「編輯系統環境變數」
    2. 點選「環境變數」→「使用者變數」→「新增」
    3. 變數名稱：ANTHROPIC_API_KEY，值：sk-ant-你的key
    4. 確定後重新開啟終端機才會生效

    方法二：PowerShell（目前工作階段）¶

    # 只對目前 PowerShell 視窗有效
$env:ANTHROPIC_API_KEY = "sk-ant-你的key"

# 確認設定成功
echo $env:ANTHROPIC_API_KEY

    方法三：PowerShell Profile（永久生效）¶

    # 開啟 PowerShell Profile 設定檔
notepad $PROFILE

# 在檔案中加入以下內容並儲存：
$env:ANTHROPIC_API_KEY = "sk-ant-你的key"

# 重新載入（或重開 PowerShell）
. $PROFILE

    第 22 章：自動化工作流 Windows 替代方案¶

    第 22 章的郵件 Agent 使用 AppleScript 讀取 Outlook，以及 launchd 定時執行，兩者都是 macOS 專用。以下是 Windows 的替代方案。

    讀取 Outlook 郵件¶

    方案一：PowerShell COM 自動化（需安裝 Outlook 桌面版）

    import subprocess

def get_emails_via_powershell():
    """使用 PowerShell COM 讀取 Outlook 郵件"""
    script = """
    $outlook = New-Object -ComObject Outlook.Application
    $namespace = $outlook.GetNamespace("MAPI")
    $inbox = $namespace.GetDefaultFolder(6)  # 6 = 收件匣
    $result = ""
    foreach ($mail in $inbox.Items | Select-Object -First 50) {
        $result += "寄件者：" + $mail.SenderName + "`n"
        $result += "主旨：" + $mail.Subject + "`n"
        $result += "---`n"
    }
    $result
    """
    result = subprocess.run(
        ["powershell", "-Command", script],
        capture_output=True, text=True, encoding="utf-8"
    )
    return result.stdout

    方案二：Microsoft Graph API（不需桌面版 Outlook，需 Azure 帳號）

    # 需安裝：pip install msal requests
import msal, requests

def get_emails_via_graph(access_token):
    """使用 Microsoft Graph API 讀取郵件"""
    headers = {"Authorization": f"Bearer {access_token}"}
    url = "https://graph.microsoft.com/v1.0/me/mailFolders/inbox/messages?$top=50"
    response = requests.get(url, headers=headers)
    return response.json().get("value", [])

    定時執行（Task Scheduler）¶

    Windows 的工作排程器對應 macOS 的 launchd：

    方法一：圖形介面

    1. 搜尋「工作排程器」開啟
    2. 右側點選「建立基本工作」
    3. 設定名稱、觸發時間（每天早上 8:00）
    4. 動作選「啟動程式」，程式設為 python，引數設為你的腳本路徑

    方法二：PowerShell 指令

    $action = New-ScheduledTaskAction -Execute "python" `
    -Argument "C:\Users\你的用戶名\email-agent\email_agent.py"

$trigger = New-ScheduledTaskTrigger -Daily -At "08:00"

Register-ScheduledTask -TaskName "EmailAgent" `
    -Action $action -Trigger $trigger -RunLevel Highest

    開啟 HTML 報告¶

    # macOS
subprocess.run(['open', report_path])

# Windows
subprocess.run(['start', report_path], shell=True)

# 跨平台寫法
import sys
if sys.platform == "darwin":
    subprocess.run(['open', report_path])
elif sys.platform == "win32":
    subprocess.run(['start', report_path], shell=True)
else:
    subprocess.run(['xdg-open', report_path])  # Linux

    其他常見差異¶

    路徑格式¶

    Claude Code 的設定檔路徑在 Windows 上：

    Claude Desktop 設定：%APPDATA%\Claude\claude_desktop_config.json
Claude Code 全域 MCP：%APPDATA%\Claude\mcp.json
Claude Code 全域 Skills：C:\Users\你的用戶名\.claude\skills\

    Python 指令¶

    macOS/Linux：python3
Windows：python（通常不區分 python 和 python3）

    換行符號¶

    Windows 預設使用 CRLF（\r\n），若遇到 Shell 腳本或 Python 檔案出現奇怪的錯誤，可在 VS Code 右下角切換換行模式為 LF。

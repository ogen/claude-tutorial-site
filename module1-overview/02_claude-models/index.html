    第二章：Claude 模型介紹¶

    模型家族（Claude 4 系列）¶

    用 Claude 一段時間後，你可能會發現同樣的問題得到的回答品質有時差很多。其中一個關鍵因素，就是你在使用哪個模型。Claude 4 系列目前有三款，各有不同定位，選對了能省時省錢，選錯了就有點可惜。

    Claude Haiku — 輕量快速型¶

    速度最快、成本最低，適合那些「不需要深度思考、就是要快」的任務。如果你在開發需要即時回應的功能，Haiku 通常是首選。

    • 特點：速度最快、成本最低
    • 適合：簡單問答、分類、摘要、即時回應需求
    • API 字串：claude-haiku-4-5-20251001

    Claude Sonnet — 均衡主力型¶

    大多數人日常用的就是這個。速度和智能的平衡點拿捏得很好，寫程式、分析報告、處理複雜對話都難不倒它。Claude.ai 預設就是 Sonnet，不需要特別設定。

    • 特點：速度與智能的最佳平衡點
    • 適合：日常開發、寫作、分析、大多數任務
    • API 字串：claude-sonnet-4-6
    • 備註：Claude.ai 預設使用此模型

    Claude Opus — 旗艦思維型¶

    需要深度推理的時候才出動。複雜的架構設計、多步驟規劃、需要高精度分析的場景，Opus 的表現明顯高出一個層次，代價是速度慢一些、費用也高一些。

    • 特點：最強推理能力，適合複雜任務
    • 適合：複雜分析、多步驟推理、高精度需求
    • API 字串：claude-opus-4-6

    功能比較¶

    三款模型的差異一目了然，選擇時主要看你對速度、推理能力、成本的優先順序：

    功能 Haiku Sonnet Opus
    速度 ⚡⚡⚡ ⚡⚡ ⚡
    推理能力 ★★ ★★★★ ★★★★★
    成本 低 中 高
    Extended Thinking ✗ ✅ ✅
    Vision（圖片分析） ✅ ✅ ✅

    Extended Thinking（擴展思考）¶

    這是一個讓 Claude 在回答之前先進行內部推理的機制。開啟後，Claude 會把思考過程跑完再給出答案，特別適合那種「答錯了代價很高」的任務。

    Extended Thinking 讓模型在回答前進行「內部推理」，適合：

    • 數學與邏輯問題
    • 複雜程式架構設計
    • 多步驟規劃任務
    • 需要高精度的分析
    啟用方式（API）：
"thinking": {
  "type": "enabled",
  "budget_tokens": 10000
}

    知識截止日期與即時搜尋¶

    Claude 的知識有個截止日期，訓練資料到某個時間點就停了，之後發生的事它不會知道。這不是缺陷，只是語言模型的特性，了解這點就不會對它抱有不合理的期待。

    • Claude 有訓練資料截止日期（knowledge cutoff）
    • Claude.ai 啟用 Web Search 後可取得即時資訊
    • Claude Code 可透過 MCP 連結即時資料來源
    • 判斷原則：靜態知識用模型記憶，即時資訊用搜尋工具

    模型選擇建議¶

    一個簡單的判斷原則：任務越複雜、越需要精確，就選越高階的模型。

    日常對話、快速回覆 → Haiku
寫程式、分析報告 → Sonnet（預設推薦）
複雜架構、深度推理 → Opus + Extended Thinking

    實作練習¶

    1. 同題不同模型：準備一個需要推理的問題（例如：「設計一個每日自動備份 PostgreSQL 資料庫並在失敗時傳送 Slack 通知的系統」），分別在 Haiku 與 Opus 得到回應，比較深度與結構差異。
    2. Extended Thinking 體驗：在 Claude.ai（Pro 方案）開啟擴展思考模式，提問一道數學或邏輯題，觀察 Claude 的「思考過程」與最終答案有何不同。
    3. 知識截止測試：詢問 Claude「你的知識截止日期是什麼時候？」，再詢問一個近期事件，觀察 Claude 如何說明不確定性並建議使用 Web Search。

    常見問題 Q&A¶

    Q：Claude.ai 我沒有選擇模型的選項，預設用哪一個？ A：Claude.ai 的預設模型是 Sonnet，它在速度與智能之間取得最佳平衡。Pro 方案可在對話中手動切換到 Opus 以處理複雜任務。

    Q：Extended Thinking 會消耗更多 token 嗎？ A：是的。Extended Thinking 需要額外的思考 token（budget_tokens），這部分會計入 API 費用。在 Claude.ai 訂閱方案中則受訊息限制約束，而非額外計費。

    Q：不同模型在同一對話中可以切換嗎？ A：在 Claude.ai 中，每次對話固定使用一個模型，切換後需開新對話。透過 API 開發時，每次呼叫都可以指定不同的 model 參數，因此可以在應用邏輯中動態選擇模型。

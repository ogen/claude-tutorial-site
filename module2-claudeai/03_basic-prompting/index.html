    第三章：基礎對話技巧與 Prompt 撰寫¶

    什麼是 Prompt？¶

    Prompt 是你傳給 Claude 的指令或問題。聽起來很簡單，但這其實是和 Claude 合作的核心技能——同樣的問題，寫法不同，得到的回答可以差很多。這一章會帶你看幾個最實用的技巧，讓你的 Prompt 從「能用」升級到「好用」。

    核心原則¶

    1. 明確說明角色與任務¶

    給 Claude 一個明確的身份和任務框架，它就能把注意力放在真正重要的地方，而不是猜你想要什麼。

    ❌ 差：幫我寫報告
✅ 好：你是一位資深財務分析師，請為非財務背景的主管撰寫
        一份 Q3 台灣半導體市場分析報告，500 字，使用條列格式

    2. 提供上下文¶

    Claude 只知道你告訴它的事情。背景資訊給得越充分，它的回答就越準確、越有針對性。

    ❌ 差：這段程式碼有什麼問題？
✅ 好：以下是一段 Python 爬蟲程式，執行時出現 ConnectionError，
        請找出原因並提供修正方案：
        [貼上程式碼]

    3. 指定輸出格式¶

    告訴 Claude 你要什麼格式的輸出，它就不會自作主張給你一大段文字，或反過來給你太簡短的答案。

    請以下列格式輸出：
- 摘要（50字以內）
- 主要發現（3點條列）
- 建議行動（2-3項）

    4. 使用 XML 標籤結構化輸入¶

    當輸入內容比較複雜，有多個不同性質的部分時，用 XML 標籤隔開會讓 Claude 更容易理解你的意圖。

    <task>分析以下文件</task>
<document>
  [文件內容]
</document>
<output_format>JSON 格式，包含 summary 和 key_points 欄位</output_format>

    上傳檔案¶

    不一定要把所有內容貼進對話框，Claude.ai 支援直接上傳檔案，讓它自己去讀：

    類型 格式 說明
    文件 PDF、DOCX、TXT 自動解析內容
    圖片 PNG、JPG、WebP 視覺分析
    程式碼 任何文字檔 語法高亮
    試算表 CSV 資料分析

    上傳方式：點選輸入框左側的迴紋針圖示，或直接拖曳檔案。

    對話管理技巧¶

    對話的管理方式也會影響回應品質。幾個實用的習慣值得養成：

    • 新對話：不同主題開新對話，避免上下文混雜
    • 繼續對話：同一主題在同一對話串延伸，Claude 會記住前文
    • 複製對話：右上角選單可複製當前對話另開分支
    • 重新生成：不滿意回應時點選重新產生，或補充「請換個方式說明」

    常用提示詞模板¶

    以下幾個模板可以直接拿去用，根據你的情況調整細節即可。

    文件摘要¶

    請摘要以下文件的重點，包含：
1. 核心主張（1句話）
2. 主要論點（3-5點）
3. 結論與建議

[貼上文件]

    程式碼審查¶

    請審查以下程式碼，重點關注：
- 潛在的 bug 或錯誤
- 效能問題
- 可讀性改善建議
請逐點說明問題並提供修正範例。

[貼上程式碼]

    角色扮演¶

    你是一位有 20 年經驗的台灣股市分析師，
熟悉台積電、聯發科等半導體族群。
我會問你關於個股與大盤的問題，
請用淺顯易懂的語言解釋，並給出具體看法。

    實作練習¶

    1. Prompt 改寫：拿以下壞 Prompt 練習改寫成好 Prompt，加入角色、格式、限制條件： 

      原版（差）：幫我寫一封道歉信
改寫目標：寄給客戶，因為延遲交貨，語氣誠懇但專業，200字以內

    2. XML 標籤結構化：選擇一份你手邊的文件（會議紀錄、文章、報告），用 <document>、<task>、<output_format> 三個 XML 標籤組織 Prompt，觀察回應品質是否提升。

    3. 對話管理實驗：開啟一個對話討論某個技術主題（如 Python 非同步程式設計），刻意在同一對話中切換到完全不相關的話題（如食譜），再切換回技術主題，觀察 Claude 是否混淆上下文，體驗「新主題開新對話」的必要性。

    常見問題 Q&A¶

    Q：Prompt 寫得越長越好嗎？ A：不一定。關鍵在於精確而非冗長。清楚說明角色、任務、格式、限制即可；不必要的描述反而可能讓 Claude 抓不到重點。一般來說，100-300 字的 Prompt 已能涵蓋大多數任務需求。

    Q：Claude 沒有按我要求的格式輸出，怎麼辦？ A：嘗試以下三種方式：① 在 Prompt 末尾加上「請嚴格按照上述格式，不要有任何額外說明」；② 提供一個具體的輸出範例（Few-Shot）；③ 使用 XML 標籤明確圈定輸出格式要求。

    Q：上傳的 PDF 有多少頁的限制？ A：Claude.ai 支援最多 100 頁的 PDF，超大文件建議分段上傳或先萃取關鍵段落。若需要處理大量文件，可改用 Claude API 的 Batch Processing 功能。

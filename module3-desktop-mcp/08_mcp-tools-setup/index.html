    第八章：連結常用外部工具（MCP）¶

    前置準備¶

    開始設定之前，先確認環境已就緒：

    node --version   # 需要 Node.js 18+
npx --version    # 隨 Node.js 附帶

    設定檔開啟方式： 

    # macOS
open ~/Library/Application\ Support/Claude/claude_desktop_config.json
# 或用 VS Code 開啟
code ~/Library/Application\ Support/Claude/claude_desktop_config.json

    Windows 用戶：設定檔路徑為 %APPDATA%\Claude\claude_desktop_config.json，以 VS Code 開啟：code "$env:APPDATA\Claude\claude_desktop_config.json"

    1. Filesystem（本地檔案存取）¶

    最常用、也最好上手的 MCP。設定完成後，Claude 就能直接讀取與寫入你指定的本地資料夾，不用再複製貼上檔案內容。

    {
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/你的用戶名/Documents",
        "/Users/你的用戶名/Desktop"
      ]
    }
  }
}

    測試：重啟 Claude Desktop 後，詢問「請列出我 Documents 資料夾中的檔案」。

    2. GitHub MCP¶

    讓 Claude 讀取 issue、PR、repo 內容，並可建立 issue、commit。如果你的工作和 GitHub 深度結合，這個整合會很有用。

    {
  "mcpServers": {
    "github": {
      "type": "http",
      "url": "https://api.githubcopilot.com/mcp",
      "headers": {
        "Authorization": "Bearer 你的_GITHUB_PAT"
      }
    }
  }
}

    取得 GitHub PAT： GitHub → Settings → Developer settings → Personal access tokens → Generate new token 所需權限：repo、read:user

    3. Google Drive¶

    讓 Claude 搜尋與讀取 Google Drive 上的文件，適合把雲端文件納入 Claude 的工作範圍。

    {
  "mcpServers": {
    "gdrive": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-gdrive"],
      "env": {
        "GDRIVE_CREDENTIALS": "/Users/你的用戶名/.config/gdrive-mcp/credentials.json"
      }
    }
  }
}

    4. Slack¶

    讓 Claude 讀取 Slack 訊息、發送通知，把 Claude 融入團隊的溝通工具。

    {
  "mcpServers": {
    "slack": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-slack"],
      "env": {
        "SLACK_BOT_TOKEN": "xoxb-你的-token",
        "SLACK_TEAM_ID": "T你的TeamID"
      }
    }
  }
}

    5. 完整設定範例¶

    同時設定多個 MCP 的話，只需要把各個 Server 都放在 mcpServers 物件裡：

    {
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/你的用戶名/Documents"
      ]
    },
    "github": {
      "type": "http",
      "url": "https://api.githubcopilot.com/mcp",
      "headers": {
        "Authorization": "Bearer ghp_你的token"
      }
    }
  }
}

    修改設定後¶

    每次修改 claude_desktop_config.json 後，需要完全重啟 Claude Desktop 才會生效： 

    macOS：Command + Q 完全退出 → 重新開啟

    常見錯誤排解¶

    如果 MCP 沒有正常連線，先看日誌是最快的方式：

    # 查看 MCP 連線日誌（macOS）
tail -f ~/Library/Logs/Claude/mcp-server-*.log

    Windows 用戶：改用 PowerShell 查看日誌：Get-Content "$env:APPDATA\Claude\Logs\mcp-server-*.log" -Wait

    錯誤訊息 可能原因 解決方式
    Server failed to start Node.js 未安裝 安裝 Node.js 18+
    Authentication failed API Key 錯誤 重新確認金鑰
    Permission denied 路徑不存在 確認資料夾路徑正確

    實作練習¶

    1. 設定 Filesystem MCP：將本章的 Filesystem 設定加入 claude_desktop_config.json，指向你的 Documents 資料夾。完全重啟 Claude Desktop 後，詢問「請列出我 Documents 資料夾的前 10 個檔案」，確認 Claude 能正確回應。

    2. GitHub MCP 整合：取得 GitHub Personal Access Token（需要 repo 和 read:user 權限），設定 GitHub MCP，然後詢問 Claude「請列出我最近 5 個 GitHub 倉庫的名稱與描述」。

    3. 錯誤排解練習：故意在設定檔中將路徑寫錯（例如指向不存在的資料夾），重啟後觀察錯誤訊息，再用 tail -f ~/Library/Logs/Claude/mcp-server-*.log 查看日誌，練習診斷問題的方法。

    常見問題 Q&A¶

    Q：修改設定檔後，為什麼 Claude Desktop 沒有反應？ A：每次修改 claude_desktop_config.json 後必須完全退出 Claude Desktop（macOS 用 Command+Q，不是只關閉視窗），再重新開啟，新設定才會生效。

    Q：GitHub PAT 的有效期限是多久？需要定期更換嗎？ A：GitHub PAT 可設定 30 天、90 天或無限期。建議設定有期限的 token，到期後在 GitHub Settings 重新產生，並更新設定檔中的 Authorization 值，再重啟 Claude Desktop 即可。

    Q：Google Drive MCP 設定比較複雜，有更簡單的替代方案嗎？ A：可以先用 Filesystem MCP，將 Google Drive 同步到本地資料夾（透過 Google Drive for Desktop），然後讓 Claude 透過 Filesystem 讀取已同步的本地副本，不需要額外設定 OAuth 憑證。

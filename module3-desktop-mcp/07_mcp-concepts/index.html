    第七章：MCP 基礎概念¶

    什麼是 MCP？¶

    MCP（Model Context Protocol） 是 Anthropic 制定的開放標準，讓 AI 模型能與外部工具、資料來源、服務進行標準化的雙向溝通。

    聽起來有點抽象，但其實只要記住一句話：

    沒有 MCP：Claude 只能閱讀你貼給它的文字
有了 MCP：Claude 可以主動去讀檔案、查資料庫、送訊息

    你不再需要把所有東西複製貼上，Claude 可以自己去取得它需要的資料。

    MCP 架構¶

    Claude Desktop 或 Claude Code 作為「客戶端（Host）」，透過 MCP 協定與各個外部工具的 Server 溝通：

    ┌─────────────────────────────────────┐
│          Claude Desktop / Code       │
│         （MCP Client / Host）        │
└──────────────┬──────────────────────┘
               │ MCP 協定
    ┌──────────┼──────────┐
    │          │          │
┌───▼───┐ ┌───▼───┐ ┌────▼────┐
│ 檔案  │ │GitHub │ │ Slack   │
│ MCP   │ │ MCP   │ │  MCP    │
└───────┘ └───────┘ └─────────┘
  本地      遠端       遠端

    傳輸類型¶

    MCP Server 有兩種運行方式，使用情境不同：

    stdio（本地標準輸入/輸出）¶

    在你的電腦本地執行，適合存取本地資源，不需要網路連線到外部服務。

    • 在你的電腦本地執行的 MCP Server
    • 透過 stdin/stdout 與 Claude 溝通
    • 適合：本地檔案存取、本地資料庫、本地腳本執行
    {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/you/Documents"]
}

    HTTP（遠端 HTTP 端點）¶

    連接到雲端服務，適合整合已有 API 的 SaaS 工具。

    • 運行在遠端伺服器的 MCP Server
    • 透過 HTTP/HTTPS 與 Claude 溝通
    • 適合：雲端服務、SaaS 工具、API 整合
    {
  "type": "http",
  "url": "https://api.example.com/mcp"
}

    設定檔位置¶

    MCP 的設定用一個 JSON 檔案管理，位置取決於你使用的產品：

    macOS Claude Desktop¶

    ~/Library/Application Support/Claude/claude_desktop_config.json

    Windows 用戶：設定檔位置為 %APPDATA%\Claude\claude_desktop_config.json，可用 code "$env:APPDATA\Claude\claude_desktop_config.json" 用 VS Code 開啟。

    設定檔結構¶

    {
  "mcpServers": {
    "server名稱": {
      "command": "執行指令",
      "args": ["參數1", "參數2"],
      "env": {
        "API_KEY": "你的金鑰"
      }
    }
  }
}

    Skills vs MCP 的差異¶

    這是很多人會混淆的地方，先把概念分清楚，後面的章節會輕鬆很多：

    項目 Skills MCP
    本質 指令集（告訴 Claude 怎麼做） 工具（給 Claude 新的能力）
    存放位置 SKILL.md 檔案 Server 程式（Node.js/Python）
    作用方式 Claude 自動載入指令 Claude 呼叫外部 API / 工具
    範例 「Git commit 要用這種格式」 「可以讀取本地檔案系統」
    兩者搭配 Skills 定義如何用 MCP 工具 MCP 提供工具，Skills 規範用法

    實作練習¶

    1. 繪製自己的 MCP 架構圖：根據本章的架構圖範例，畫出你想建立的 MCP 環境。例如：Claude Desktop 作為 Host，連結 filesystem（本地文件）和 GitHub（程式碼倉庫）兩個 Server。

    2. 設定檔結構練習：不實際執行，先手動撰寫一份 claude_desktop_config.json，設定一個名為 my-notes 的 filesystem MCP，指向你的筆記資料夾路徑。檢查 JSON 語法是否正確（可貼入 jsonlint.com 驗證）。

    3. 概念釐清測驗：判斷以下各項屬於 Skills 還是 MCP：

    4. 「Claude 每次 commit 都要寫繁體中文訊息」→ ___
    5. 「Claude 可以查詢 Notion 資料庫」→ ___
    6. 「Claude 做程式碼審查時要按照這個 checklist」→ ___
    7. 「Claude 可以發送 Slack 訊息」→ ___

    常見問題 Q&A¶

    Q：stdio 和 HTTP 兩種傳輸方式，初學者應該先學哪一種？ A：先從 stdio 開始。stdio MCP 在本地執行，不需要額外的伺服器或網路設定，適合存取本地檔案、資料夾；HTTP 類型適合連接已有 MCP 端點的雲端服務（如 GitHub）。

    Q：MCP 設定檔寫錯會怎樣？Claude Desktop 會當機嗎？ A：不會當機，但該 MCP Server 不會啟動，Claude Desktop 會在 Developer 設定頁面顯示連線錯誤。可以透過 tail -f ~/Library/Logs/Claude/mcp-server-*.log 查看詳細錯誤訊息。

    Q：可以同時連結多個 MCP Server 嗎？ A：可以，mcpServers 是一個 JSON 物件，可以包含任意數量的 Server 設定，每個 Server 用不同的 key 命名即可。

    第二十四章：Claude 企業方案總覽¶

    為什麼企業需要專屬方案？¶

    個人用 Claude Pro 解決的是「一個人的效率問題」，但當整個組織都要用 Claude，問題就複雜很多了：誰能用？用到什麼程度？資料有沒有留存？員工離職了怎麼辦？這些都不是 Pro 方案能回答的問題。

    Claude 的 Team 和 Enterprise 方案，就是為了解決這些「組織層面」的需求而設計的。

    Team vs Enterprise 比較¶

    兩個方案的出發點不同：Team 是讓小型團隊共用更多訊息額度，Enterprise 是讓大型組織能夠完整管控、符合法規：

    功能 Pro（個人） Team Enterprise
    月費 $20/人 $25/人 洽談
    最低人數 1 人 2 人 通常 25 人+
    訊息額度 較高 更高 自訂
    Projects ✅ ✅ ✅
    Claude Code ✅ ✅ ✅
    管理控制台 ✗ ✅（基本） ✅（完整）
    SSO（單一登入） ✗ ✗ ✅
    SCIM 用戶佈建 ✗ ✗ ✅
    稽核日誌 ✗ ✗ ✅
    自訂資料保留政策 ✗ ✗ ✅
    優先客戶支援 ✗ ✗ ✅
    專屬帳號管理 ✗ ✗ ✅

    管理控制台（Admin Console）¶

    Enterprise 方案的核心是管理控制台，讓 IT 管理員能集中管理整個組織的 Claude 使用情況。

    主要功能¶

    使用者管理 - 新增、停用、刪除帳號 - 依部門或角色分組 - 批次匯入（CSV 或透過 SCIM）

    權限設定 - 控制哪些用戶能使用 Claude Code - 設定 Projects 的共用範圍 - 限制特定功能的存取（如 Web Search）

    使用量監控 - 各部門的訊息使用量統計 - 活躍用戶數追蹤 - 異常使用警報

    進入管理控制台¶

    claude.ai → 右上角帳號圖示 → Admin Console
（需要具備管理員權限的帳號才能看到此選項）

    SSO 與 SCIM 整合¶

    大型企業通常已有自己的身份驗證系統（如 Okta、Azure AD、Google Workspace），SSO 讓員工用現有的公司帳號登入 Claude，不需要額外設定密碼。

    支援的 SSO 標準¶

    • SAML 2.0：最廣泛支援的企業 SSO 標準
    • OIDC（OpenID Connect）：現代 OAuth 2.0 延伸協定

    SCIM 自動化用戶佈建¶

    SCIM 讓用戶帳號管理完全自動化：

    員工入職 → HR 系統新增帳號
    ↓
SCIM 同步到 Claude
    ↓
員工自動獲得 Claude 存取權限

員工離職 → HR 系統停用帳號
    ↓
SCIM 自動撤銷 Claude 存取
    ↓
不需要 IT 手動操作

    設定 SSO 的基本流程¶

    1. 在管理控制台開啟 SSO 設定頁面
    2. 下載 Claude 的 SAML metadata XML
    3. 在你的 IdP（如 Okta）建立新的 SAML 應用程式
    4. 填入 Claude 提供的 ACS URL 和 Entity ID
    5. 完成設定後進行測試登入

    實際操作細節依 IdP 不同而異，Anthropic 企業支援團隊會提供對接指引。

    使用政策與治理¶

    拿到 Enterprise 方案不等於就可以讓員工自由使用，還需要建立內部的使用政策。幾個值得提前規劃的面向：

    使用場景白名單¶

    明確告訴員工 Claude 適合用在哪些場景，例如： - ✅ 撰寫對外文件、郵件草稿 - ✅ 程式碼開發與 Debug - ✅ 會議記錄整理 - ⚠️ 客戶個資相關任務（需先確認合規） - ❌ 公司機密財務數字（未啟用零資料保留前）

    分階段導入建議¶

    不建議一次讓全公司都開始用，容易失控：

    第一階段（1-2 週）：IT + 少數種子用戶試用，建立使用規範
第二階段（1 個月）：部門主管先行，收集反饋
第三階段（3 個月）：全員開放，搭配教育訓練

    實作練習¶

    1. 方案評估：根據你所在組織的規模和需求，填寫以下評估表，判斷適合 Team 還是 Enterprise：
    評估項目 你的情況
    預計使用人數
    是否需要 SSO 整合
    是否有資料合規要求
    是否需要稽核日誌

    1. 管理控制台探索：若已有 Team 方案，進入管理控制台，列出目前有哪些活躍用戶、各自的使用量，以及哪些功能尚未啟用。

    2. 使用政策草稿：參考本章的使用場景建議，為你的組織起草一份 Claude 使用政策，至少包含：適用範圍、允許的使用場景、禁止的使用場景、資料處理規範。

    常見問題 Q&A¶

    Q：Team 和 Enterprise 的訊息額度差多少？ A：Team 方案的額度約為 Pro 的 2-3 倍；Enterprise 的額度可依需求客製化，甚至可以購買無限額度。具體數字需透過 Anthropic 業務洽談，因組織規模與使用情境而異。

    Q：SSO 設定完成後，員工還需要個別建立 Claude 帳號嗎？ A：啟用 SSO 後，員工直接用公司帳號（如 Google Workspace 或 Okta）登入 Claude，不需要另外建立帳號。若同時啟用 SCIM，帳號的建立和刪除也會完全自動化。

    Q：Enterprise 方案的最低合約期是多久？ A：通常是一年期合約，具體條款依洽談結果而定。若你的組織還在評估階段，可以先試用 Team 方案，確認需求後再升級。

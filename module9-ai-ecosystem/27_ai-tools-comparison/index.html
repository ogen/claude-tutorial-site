    第二十七章：AI 工具生態全覽與比較¶

    為什麼要了解整個生態？¶

    現在市面上的 AI 工具多到讓人眼花撩亂，但它們不是彼此替代的關係，而是各有側重的工具。就像你不會只用一種程式語言、只用一種框架，AI 工具組合也是同樣的邏輯——選對工具放在對的位置，效率才能真正提升。

    這章會帶你完整瀏覽目前主流的 AI 工具，讓你知道每個工具最擅長什麼、最不適合什麼，以及和 Claude 搭配使用的可能性。

    主流工具一覽¶

    Claude（Anthropic）¶

    我們整份教學的主角。Claude 最突出的特質是長文本處理能力、指令遵循的精確度，以及在寫作、分析、複雜推理上的表現。Claude Code 讓它成為開發者的有力工具，Skills 系統則讓工作流標準化。

    • 最強項：複雜分析、長文件處理、精確遵循指令、程式碼開發（Claude Code）
    • 相對弱項：即時資訊（需搭配 Web Search）、圖像生成（不支援）
    • 定價：Free / Pro $20 / Team $25 / Enterprise 洽談

    ChatGPT（OpenAI）¶

    最廣為人知的 AI 助手，GPT-4o 是目前的主力模型。ChatGPT 的生態系最成熟，Plugin / GPT Store 的第三方整合最多，DALL-E 圖像生成也是內建功能。

    • 最強項：多模態（圖像生成與分析）、廣大的第三方整合生態、語音對話
    • 相對弱項：長文本一致性、指令遵循的穩定度有時不如 Claude
    • 定價：Free / Plus $20 / Team $25 / Enterprise 洽談

    Gemini（Google）¶

    Google 的 AI 助手，最大優勢是與 Google 生態（Gmail、Docs、Drive、Search）的深度整合。Gemini 1.5 Pro 的 context window 達到 100 萬 token，處理超長文件時有優勢。

    • 最強項：Google Workspace 整合、超長 context window、即時搜尋（原生）
    • 相對弱項：指令遵循的細緻度、企業資安合規
    • 定價：Free / Advanced $20 / Google One AI Premium 方案

    GitHub Copilot（Microsoft / OpenAI）¶

    專門為開發者設計的 AI 輔助工具，以 IDE 內行內程式碼補全為核心功能。它不是對話型助手，而是一邊寫程式一邊給建議的「副駕駛」。

    • 最強項：IDE 內即時程式碼補全、PR 自動摘要、多種 IDE 支援
    • 相對弱項：不適合複雜的架構討論或長篇分析
    • 定價：Individual $10 / Business $19 / Enterprise $39（每人每月）

    Cursor¶

    Cursor 是基於 VS Code 深度改造的 AI-first IDE，底層可以選擇使用 Claude 或 GPT-4 作為模型。它的 Composer 功能讓 AI 能同時編輯多個檔案，比 GitHub Copilot 的補全更進一步。

    • 最強項：多檔案同時編輯（Composer）、完整 VS Code 相容性、可選 Claude 作為後端
    • 相對弱項：不是對話型助手，不適合分析或寫作任務
    • 定價：Hobby 免費 / Pro $20 / Business $40（每人每月）

    各工具的最適場景¶

    與其問「哪個最好」，不如問「這個任務用哪個最合適」：

    任務類型 推薦工具 原因
    長篇文件分析、報告撰寫 Claude 長文本一致性強、指令遵循精確
    程式碼補全（邊寫邊用） Copilot / Cursor 深度 IDE 整合，即時反應
    複雜多檔案重構 Claude Code / Cursor 多檔案編輯能力
    需要生成圖像 ChatGPT（DALL-E） 圖像生成內建
    Google Workspace 任務 Gemini 原生整合 Gmail、Docs
    即時資訊查詢 ChatGPT / Gemini 即時搜尋能力較強
    建立自動化工作流 Claude（API + Skills） MCP、Skills 生態完整
    企業合規與安全 Claude Enterprise 合規認證最完整

    實作練習¶

    1. 用例分類練習：列出你過去一週使用 AI 工具的 10 個場景，對照本章的最適場景表，判斷每個場景是否用了最合適的工具。

    2. 工具試用比較：選擇一個你常做的任務（如：寫一段 Python 爬蟲、分析一份文件），分別用 Claude 和另一個工具（ChatGPT 或 Gemini）完成，比較輸出品質、回應速度、操作體驗的差異。

    3. 情境選型決策：根據以下三種情境，各選出最合適的工具組合並說明理由：

    4. 前端工程師，每天需要快速補全程式碼、偶爾做架構討論
    5. 內容行銷，每週需要撰寫部落格文章、製作配圖、分析競品
    6. 後端工程師，需要處理複雜的系統設計、多模組重構、API 文件撰寫

    常見問題 Q&A¶

    Q：Claude 和 ChatGPT 到底哪個比較強？ A：這個問題沒有標準答案，取決於任務類型。Claude 在長文本分析、精確遵循複雜指令、以及開發工作流（Claude Code + Skills）上普遍表現較好；ChatGPT 在多模態（圖像生成/分析）、第三方工具整合生態上更豐富。建議兩個都試用，根據你最常做的任務決定主力工具。

    Q：Cursor 和 Claude Code 的定位相同嗎？ A：不完全相同。Cursor 是一個完整的 IDE（基於 VS Code），強調在編輯器內的 AI 輔助體驗；Claude Code 是命令列工具，強調完整的工作流自動化（讀取、修改、執行、commit）。Cursor 適合習慣 GUI 的開發者，Claude Code 適合喜歡在終端機中操作的開發者。兩者也可以同時使用，並不衝突。

    Q：付費同時使用多個 AI 工具划算嗎？ A：看你的使用頻率。如果每個工具你都有明確的高頻使用場景，同時訂閱是值得的。但如果只是偶爾用一下，大多數工具的免費版已能滿足輕度需求。第二十九章會介紹如何根據使用量設計最划算的工具組合。

    第二十九章：建立你的個人 AI 工具組合¶

    與其追求「最強工具」，不如找到「最適組合」¶

    AI 工具更新很快，每隔幾個月就有新的「最強模型」出現。如果每次都跟著換工具，你花在遷移和重新學習上的時間，可能比工具本身幫你省下的時間還多。

    比較好的思維方式是：先清楚自己的工作場景，再選對應工具，最後固定下來持續優化。這一章會給你一個選型框架，幫你建立屬於自己的 AI 工具組合。

    選型框架：四個問題¶

    在決定要不要加入某個 AI 工具之前，先回答這四個問題：

    1. 這個工具解決了我哪個具體的工作痛點？¶

    如果答案是「感覺很厲害」或「大家都在用」，那先不要訂閱。只有當你能具體說出「它幫我省掉了 X 這件每天都要做的事」，才值得付費。

    2. 我目前已有的工具能做到嗎？¶

    新工具的功能有 80% 都和你已有的工具重疊，剩下的 20% 才是真正的差異化優勢。評估時要問的是那 20%，而不是整體功能清單。

    3. 月費和節省的時間成本相比如何？¶

    粗略估算：如果你的時薪是 1000 元台幣，一個工具每天幫你省 30 分鐘，一個月就節省了約 10,000 元價值的時間。相比之下，$20 美元的月費（約 640 台幣）很划算；如果只省 5 分鐘，就不一定值得了。

    4. 這個工具有沒有學習成本？遷移成本？¶

    有些工具功能很強，但要花一兩週才能上手。如果你平均每 3 個月就換一次工具，這個學習成本會讓你一直停在入門階段。

    依職能的工具組合建議¶

    後端 / 全端工程師¶

    核心需求：程式碼開發、架構設計、Debug、文件

    推薦組合：
Claude Pro（$20）   → 主力：架構討論、複雜 Debug、文件撰寫
Claude Code          → 包含在 Pro 中：工作流自動化、多檔案重構
GitHub Copilot（$10）→ 日常補全：在 IDE 中邊寫邊用

總費用：約 $30/月

選配（若預算允許）：
Cursor Pro（$20）   → 替代或搭配 Copilot，GUI 體驗更好

    工作流設計： - 需求討論、架構設計 → Claude（對話） - 日常寫程式 → Copilot 補全 + 偶爾問 Copilot Chat - 複雜問題、大規模修改 → Claude Code - PR Review → Claude Code + Copilot for PRs

    前端工程師¶

    核心需求：UI 實作、程式碼品質、設計還原

    推薦組合：
Claude Pro（$20）       → 主力：UI 設計討論、複雜問題
Claude Code              → 包含在 Pro 中
Frontend Design Skill    → 免費：讓 Claude 產生有設計感的 UI
Cursor Pro（$20）        → Composer 多檔案編輯 CSS/JS/TSX

總費用：約 $40/月

    工作流設計： - 設計稿轉程式碼 → 截圖上傳給 Claude，搭配 Frontend Design Skill - 多個相關元件同時修改 → Cursor Composer - TypeScript 型別問題 → Copilot Chat 快速問

    內容創作 / 文案行銷¶

    核心需求：寫作、研究、圖像、多平台發佈

    推薦組合：
Claude Pro（$20）   → 主力：長文撰寫、品牌聲音維持、研究分析
ChatGPT Plus（$20）→ 圖像生成（DALL-E）、語音輸入輸出

總費用：約 $40/月

若預算有限，可先只訂 Claude Pro，圖像生成用 Canva AI 等免費方案替代

    工作流設計： - 長篇文章、策略文件 → Claude（長文一致性更好） - 快速生成配圖 → ChatGPT（DALL-E） - 每日社群貼文 → 建立 Claude Project 存放品牌指南，維持語氣一致

    資料分析師 / 研究員¶

    核心需求：資料理解、報告撰寫、程式碼（Python/SQL）

    推薦組合：
Claude Pro（$20）   → 主力：長報告分析、研究整理、Python 協助
Gemini Advanced（$20）→ Google Sheets / Looker Studio 原生整合

若組織使用 Microsoft 365：
Copilot for M365（$30）→ Excel / PowerPoint / Teams 深度整合

    工作流設計： - 上傳 PDF 報告、CSV 資料 → Claude 直接分析 - 需要操作 Google Sheets → Gemini - 需要操作 Excel / PowerPoint → Copilot for M365

    個人學習 / 輕度使用¶

    核心需求：日常問答、學習輔助、偶爾寫文

    推薦組合：
Claude Free（免費）  → 日常問答，有訊息限制但夠用
ChatGPT Free（免費）→ 補充：圖像分析、語音
Gemini Free（免費） → 補充：Google 相關任務

若要升級，優先升 Claude Pro（最全能）

    成本優化策略¶

    善用免費層¶

    大多數工具的免費版對輕度使用者已經夠用，在升級前先評估你的實際使用量：

    工具 免費版限制 何時值得升級
    Claude 每日訊息有限制 每天用到限制、需要 Projects 或 Claude Code
    ChatGPT GPT-4o 有限次數 需要大量圖像生成、需要 GPT-4 的更高額度
    Gemini 每日限制 深度使用 Google Workspace 整合
    Copilot 每月 2000 次補全 每天寫程式超過 2 小時

    避免功能重疊的浪費¶

    同時訂閱多個工具，但每個都只用到 20% 的功能，是最常見的浪費：

    常見的冗餘組合（不推薦）：
Claude Pro + ChatGPT Plus + Gemini Advanced
→ 三個都有對話功能，大量重疊
→ 除非你有明確的不同用途，否則只需要一個主力

更好的組合思路：
一個「全能對話助手」+ 一個「專業開發工具」
Claude Pro + GitHub Copilot   ← 對話 + 行內補全
或
Claude Pro + Cursor Pro       ← 對話 + AI-first IDE

    用量監控¶

    定期看一下你實際用了多少，避免訂而不用：

    # 追蹤 Claude API 用量（若你有 API 呼叫的話）
import anthropic

client = anthropic.Anthropic()

# 在 Anthropic Console 的 Usage 頁面可以看到：
# - 每日 Token 消耗量
# - 各模型的使用比例
# - 費用趨勢圖

    工具組合的演進路徑¶

    工具組合不是一次決定就不變的，建議按以下節奏演進：

    第 1-3 個月：探索期
  → 只用免費版，找出你的高頻使用場景
  → 觀察哪些任務因為沒有 AI 工具而效率低

第 3-6 個月：確立期
  → 針對最高頻的場景付費訂閱
  → 建立穩定的工作流（Prompts、Skills、CLAUDE.md）

第 6 個月後：優化期
  → 每季評估一次，檢視哪些工具有在使用
  → 逐步加入第二個工具（若有需要）
  → 開始自建 API 工具（若頻繁使用相同流程）

    實作練習¶

    1. 現有工具盤點：列出你目前在用的所有 AI 工具（包含免費版），以及各自的月費。對每個工具寫下「我上週用它做了什麼」，沒有使用記錄的就考慮取消訂閱。

    2. 個人工具組合設計：根據本章的選型框架和職能建議，設計你的理想工具組合。包含：工具名稱、月費、主要用途、與其他工具的分工。

    3. 30 天試驗：選擇你目前最常用但體驗不滿意的一個 AI 工具，用本章推薦的組合策略替換看看，30 天後記錄效率變化和感受。

    常見問題 Q&A¶

    Q：AI 工具更新這麼快，今天選的組合明年還適用嗎？ A：工具會變，但工作場景不會變太快。你今天需要的「協助寫長報告」「幫我 Debug 複雜問題」「快速補全程式碼」，一年後大概還是同樣的需求。所以以工作場景為基礎選工具，比追新功能穩定很多。當某個工具的某個功能對你特別重要時，再考慮切換。

    Q：公司已經買了 Microsoft Copilot for M365，還需要自己訂 Claude 嗎？ A：Copilot for M365 非常適合在 Office 套件（Word、Excel、Teams）中使用，但在長文本分析、開發任務、複雜推理上，Claude 通常表現更好。如果你的工作大量依賴 Office，M365 Copilot 很夠用；如果你有開發需求或需要深度分析，加訂 Claude Pro 會有明顯補充。

    Q：AI 工具訂閱費用可以報公帳嗎？ A：大多數情況可以，但需要確認公司政策。Claude Pro / Team 的帳單可以在 claude.ai 的 Settings → Billing 下載 PDF 發票，標明是 Anthropic 的服務費用，通常符合軟體工具的費用報銷類別。Enterprise 方案可以開立統一發票或公司帳單。

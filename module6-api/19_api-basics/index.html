    第十九章：基礎 API 呼叫¶

    Python 基礎範例¶

    最簡單的呼叫方式，只需要幾行程式碼：

    import anthropic

client = anthropic.Anthropic()

# 基本呼叫
response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "請用一句話說明什麼是 MCP"}
    ]
)

print(response.content[0].text)

    System Prompt 設計¶

    System Prompt 讓你在對話開始前就給 Claude 一個角色和框架，後續所有的對話都會在這個設定下進行：

    response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    system="""你是一位專業的台股分析師，
    熟悉台積電、聯發科等半導體族群。
    回答時使用繁體中文，語氣專業但易懂。
    不確定的資訊請明確說明，不要猜測。""",
    messages=[
        {"role": "user", "content": "台積電最近的法說會重點是什麼？"}
    ]
)

    多輪對話¶

    Claude API 本身是無狀態的，它不會自己記住上一次說了什麼。要實現多輪對話，你需要把完整的對話歷史一起傳進去：

    conversation_history = []

def chat(user_message):
    conversation_history.append({
        "role": "user",
        "content": user_message
    })

    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        system="你是一個 Python 程式設計助手",
        messages=conversation_history
    )

    assistant_message = response.content[0].text
    conversation_history.append({
        "role": "assistant",
        "content": assistant_message
    })

    return assistant_message

# 使用
print(chat("我想學習 Python 的 list comprehension"))
print(chat("能給我一個實際的例子嗎？"))

    Streaming 串流輸出¶

    等待 Claude 把完整回應算完再顯示，有時候要等好幾秒。Streaming 讓文字即時出現，使用者體驗好很多：

    with client.messages.stream(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{"role": "user", "content": "請寫一首關於台灣的詩"}]
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)

    Node.js 範例¶

    如果你的專案是 Node.js，用法基本一樣，只是語法不同：

    import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();

const response = await client.messages.create({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  messages: [{ role: "user", content: "Hello, Claude!" }],
});

console.log(response.content[0].text);

    實作練習¶

    1. 第一個 API 呼叫：複製本章的 Python 基礎範例，修改 content 為一個你感興趣的問題（建議用中文），執行並確認回應正常。觀察 response 物件的結構，嘗試印出 response.usage（token 使用量）。

    2. 建立多輪對話聊天機器人：複製本章的多輪對話範例，用一個 while 迴圈讓使用者持續輸入，實作一個可以在終端機持續對話的 CLI 聊天機器人： 

      while True:
    user_input = input("你：")
    if user_input == "quit":
        break
    print(f"Claude：{chat(user_input)}")

    3. Streaming 體驗：將基礎呼叫改成 Streaming 模式，請 Claude 「寫一首關於台灣的五言絕句」，觀察文字逐字出現的效果，感受 Streaming 對使用者體驗的影響。

    常見問題 Q&A¶

    Q：max_tokens 設太小會怎樣？ A：Claude 的回應會在達到 max_tokens 時強制截斷，可能導致輸出不完整（例如程式碼寫到一半）。一般建議設 1024-4096，需要長篇輸出（如完整文章）時可設到 8192。注意：設定越高並不代表每次都消耗那麼多 token，只消耗實際輸出的量。

    Q：多輪對話的 conversation_history 要一直保存嗎？ A：是的，Claude API 本身是無狀態的，每次呼叫都需要傳入完整的對話歷史。這表示對話越長，每次 API 呼叫的 input token 越多，費用也越高。長對話建議定期壓縮歷史（只保留摘要）或設定最大輪數限制。

    Q：Streaming 模式和一般模式費用一樣嗎？ A：完全一樣。Streaming 只影響回應「顯示」的方式（即時串流 vs 等待完成），不影響計費方式，都是以實際消耗的 input + output tokens 計費。

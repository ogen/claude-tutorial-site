    第十一章：Claude Code + IDE 整合¶

    VS Code 整合¶

    喜歡在 IDE 裡工作的人有個好消息：Claude Code 有官方的 VS Code 擴充套件，讓你不用離開編輯器就能和 Claude 對話。

    安裝擴充套件¶

    1. 開啟 VS Code
    2. 按 Cmd+Shift+X 開啟擴充功能市集（Windows 為 Ctrl+Shift+X）
    3. 搜尋「Claude Code」
    4. 安裝 Anthropic 官方套件

    使用方式¶

    安裝後，在 VS Code 中可以： - 選取程式碼 → 右鍵 → Ask Claude - 使用 Cmd+Shift+P（Windows 為 Ctrl+Shift+P）→ 輸入 Claude 相關指令 - 側邊欄出現 Claude Code 對話面板

    常用操作¶

    選取程式碼後，右鍵選單會出現幾個最常用的操作：

    選取程式碼後：
- Explain this code（解釋程式碼）
- Fix this code（修正錯誤）
- Write tests for this（寫測試）
- Refactor this（重構）

    JetBrains 整合（IntelliJ / PyCharm / WebStorm）¶

    JetBrains 系列的整合方式類似，透過 Plugin 安裝：

    安裝外掛¶

    1. Settings → Plugins → Marketplace
    2. 搜尋「Claude Code」
    3. 安裝並重啟 IDE

    終端機整合（推薦：iTerm2 / Windows Terminal）¶

    如果你習慣在終端機工作，搭配 iTerm2（macOS）的分割視窗效果最好——一邊看程式碼，一邊和 Claude 對話，兩者都不用離開。

    Windows 用戶：推薦使用 Windows Terminal（Microsoft Store 免費下載），同樣支援分割視窗，使用體驗和 iTerm2 相近。

    # 建議的工作區配置
# 左側：你的程式碼（VS Code 或 vim）
# 右側：Claude Code 對話視窗

    工作流示範：修 Bug¶

    一個真實的修 Bug 流程大概長這樣：

    $ claude
> 我的 Python 爬蟲在執行到第 50 頁時會當掉，
  錯誤訊息如下：

  requests.exceptions.ConnectionError: 
  HTTPSConnectionPool(host='example.com', port=443): 
  Max retries exceeded

Claude: 我來看一下你的爬蟲程式碼...
[Claude 讀取 scraper.py]

這個錯誤通常是因為請求太頻繁被伺服器封鎖。
建議修改方案：
1. 加入隨機 delay（1-3秒）
2. 加入 retry 機制
3. 輪換 User-Agent

要我直接修改 scraper.py 嗎？

> 是的，請修改
[Claude 直接修改檔案]

    實作練習¶

    1. 安裝 VS Code 擴充套件：在 VS Code 安裝 Claude Code 官方擴充套件，選取一段你熟悉的程式碼，用右鍵選單的「Explain this code」功能讓 Claude 解釋，確認側邊欄的對話面板是否正常運作。

    2. 修 Bug 工作流：在 VS Code 中刻意在一個函數裡寫入一個明顯的 Bug（例如：除以零、錯誤的變數名稱），選取該程式碼後用 Claude 的「Fix this code」功能，觀察 Claude 如何診斷並修正。

    3. 分割視窗開發：在 iTerm2（或 macOS 終端機）設定分割視窗：左側用 VS Code 開啟你的專案，右側開啟 Claude Code CLI（claude）。試著用右側的 Claude 說明後讓它直接修改左側的檔案，體驗真正的 AI 輔助開發流程。

    常見問題 Q&A¶

    Q：CLI（終端機）和 VS Code 擴充套件，應該用哪一個？ A：兩者各有優勢。VS Code 擴充套件適合針對特定程式碼片段的快速操作（解釋、修正、測試）；CLI適合需要跨多個檔案、執行指令、完整工作流的任務。建議搭配使用：大任務用 CLI，小修正用擴充套件。

    Q：在 IDE 使用 Claude Code 需要重新登入嗎？ A：不需要，IDE 擴充套件與 CLI 共用同一份登入狀態（儲存在 ~/.claude/ 目錄），安裝後應該會自動偵測已有的登入。若顯示未登入，重啟 VS Code 或在終端機執行 claude 重新授權即可。

    Q：JetBrains 系列（PyCharm、IntelliJ）的體驗和 VS Code 一樣嗎？ A：功能基本相同，但 JetBrains 外掛的界面整合在右側的 Claude 面板，而非右鍵選單。PyCharm 用戶可以直接在外掛面板中貼入程式碼詢問，不需要先選取。

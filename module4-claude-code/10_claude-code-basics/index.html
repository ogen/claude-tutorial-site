    第十章：Claude Code 基礎操作與 CLAUDE.md¶

    常用指令速查¶

    先把這張速查表收起來，用到的時候回來查：

    指令 說明
    claude 啟動互動模式
    claude -p "..." 單次執行
    claude --continue 繼續上次對話
    /help 顯示說明
    /clear 清除對話記憶
    /status 顯示目前狀態
    /mcp 查看 MCP 連線狀態
    Ctrl+C 中斷目前操作

    Claude Code 工作流程¶

    四階段工作法¶

    使用 Claude Code 最有效的方式，是把工作分成四個明確的階段，讓 Claude 在每個階段都知道它該做什麼：

    1. Explore（探索）
   Claude 閱讀你的程式碼與資料夾結構

2. Plan（規劃）
   Claude 提出實作計畫，等你確認

3. Implement（實作）
   Claude 實際修改程式碼

4. Commit（提交）
   Claude 協助 git commit

    實際操作範例¶

    一個完整的開發流程大概長這樣：

    $ claude
> 我想為這個 Express.js 應用加入 JWT 認證，請先了解現有結構

[Claude 閱讀 package.json、路由檔案等]

Claude: 我看到你的專案使用 Express 4.x，路由在 /routes 資料夾...
        建議的實作計畫如下：
        1. 安裝 jsonwebtoken、bcryptjs
        2. 建立 middleware/auth.js
        3. 修改 routes/users.js 加入登入端點
        是否開始實作？

> 好，開始吧

    CLAUDE.md 設定檔¶

    CLAUDE.md 是放在專案根目錄的設定檔，讓 Claude Code 在每次對話開始時自動載入專案背景知識。有了它，你不用每次都重新介紹「這個專案是做什麼的」、「要注意哪些規則」，Claude 一進來就知道了。

    建立 CLAUDE.md¶

    touch CLAUDE.md

    Windows 用戶：改用 New-Item CLAUDE.md -ItemType File（或縮寫 ni CLAUDE.md）。

    CLAUDE.md 範本¶

    # 專案說明

## 專案名稱
Email Agent — 每日郵件摘要自動化工具

## 技術架構
- 語言：Python 3.11
- 郵件存取：AppleScript + Outlook for Mac
- 輸出格式：HTML 報告
- 執行環境：macOS

## 重要規則
- 所有程式碼使用繁體中文註解
- HTML 輸出檔案放在 /output 資料夾
- API Key 存在 .env 檔，不要 commit

## 常用指令
- 執行主程式：`python main.py`
- 測試：`pytest tests/`
- 查看輸出：`open output/report.html`

## 目錄結構
project/
├── main.py          # 主程式
├── agents/          # Agent 模組
├── templates/       # HTML 模板
├── output/          # 輸出報告
└── tests/           # 測試檔案

    CLAUDE.md 放置位置¶

    CLAUDE.md 可以放在不同層級，作用範圍也不同：

    全域設定：~/.claude/CLAUDE.md     （所有專案共用）
專案設定：./CLAUDE.md              （僅此專案）
子目錄：  ./src/CLAUDE.md          （僅 src 目錄）

    Windows 用戶：全域 CLAUDE.md 路徑為 C:\Users\你的用戶名\.claude\CLAUDE.md，~/ 在 PowerShell 中等同於 $HOME\。

    實作練習¶

    1. 建立專案 CLAUDE.md：為你手邊的任何一個專案（或練習用的空專案）建立 CLAUDE.md，至少包含：專案說明、主要技術架構、重要規則（如程式碼風格）、常用指令。建立後執行 claude，詢問「這個專案的主要架構是什麼？」確認 Claude 是否正確載入了 CLAUDE.md。

    2. 四階段工作法實戰：選擇一個小任務（如「為 main.py 加入 logging 功能」），刻意使用四階段流程：

    3. 先讓 Claude Explore（「請先閱讀 main.py 和相關檔案」）
    4. 再讓 Claude Plan（「請提出實作計畫」）
    5. 確認計畫後再 Implement（「好，開始實作」）

    6. 最後 Commit（「幫我寫 commit message」）

    7. 指令速查練習：在 Claude Code 對話中依序執行 /help、/status、/mcp，閱讀每個指令的輸出，了解目前工作狀態與可用工具。

    常見問題 Q&A¶

    Q：CLAUDE.md 和 Claude.ai 的 Custom Instructions 有什麼不同？ A：Custom Instructions 是個人偏好設定，全平台生效；CLAUDE.md 是專案層級的背景知識，只在 Claude Code 讀取該專案時生效，適合放「這個專案特有」的規則、架構、指令，讓 Claude 每次都能快速進入狀況。

    Q：什麼時候應該用 /clear 清除對話？ A：當你切換到完全不同的任務時（例如從修 Bug 切換到新增功能），建議執行 /clear 清除前一個任務的 context，避免舊資訊干擾新任務的判斷。同一任務的延伸討論則不需要清除。

    Q：Claude Code 修改了錯誤的檔案怎麼辦？ A：Claude Code 的每次修改都是真實的檔案操作，因此強烈建議在使用前確保專案已 git init 並有 commit 記錄，這樣可以用 git diff 查看改動，用 git checkout 還原錯誤修改。
